230. Kth Smallest Element in a BST

Given a binary search tree, write a function kthSmallest to find the kth smallest element in it.

Note:

You may assume k is always valid, 1 ≤ k ≤ BST’s total elements.

Follow up:

What if the BST is modified (insert/delete operations) often and you need to find the kth smallest frequently? How would you optimize the kthSmallest routine?

Hint:

1. Try to utilize the property of a BST.
2. What if you could modify the BST node's structure?
3. The optimal runtime complexity is O(height of BST).

Update (2015-02-12):

For C programmers: Try to solve it in-place in O(1) space.

Solution 1:

Modify the BST, move current node and its child tree to current node’s left child node’s right most child node, make the BST to be a line.

Code

Solution 2:

DFS binary search, count all child node of current node’s left child. If k <= count mean target node is in current node’s left child tree, else k > count + 1 means k is larger than number of left child tree, so target node is in the right of the target tree.
